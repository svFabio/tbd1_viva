# OBJETIVOS

## Objetivo General
Diseñar e implementar una base de datos relacional robusta y escalable que permita gestionar y centralizar de manera eficiente la información del ecosistema digital de VIVA (Nuevatel PCS de Bolivia S.A.), abarcando su infraestructura operativa, segmentos de clientes (individuales y corporativos), servicios de conectividad y programas de fidelidad.

## Objetivos Específicos
1. **Analizar el dominio** del ecosistema digital y de telecomunicaciones de VIVA para identificar los requerimientos de información, reglas de negocio y restricciones del sistema.
2. **Diseñar el modelo conceptual (Entidad-Relación)** que represente de forma lógica e intuitiva las entidades clave del negocio, tales como clientes (prepago, postpago, corporativos), planes de telefonía e internet, plataformas digitales y programas de fidelización.
3. **Desarrollar el modelo lógico** aplicando reglas de normalización para asegurar la integridad relacional de los datos, minimizar la redundancia y optimizar las estructuras de almacenamiento.
4. **Implementar el modelo físico** utilizando un Sistema Gestor de Base de Datos (SGBD) adecuado, definiendo los tipos de datos óptimos, llaves primarias, llaves foráneas e índices.
5. **Establecer políticas de seguridad y control de acceso** mediante la creación de roles (ej. administrador, atención al cliente, facturación) y asignación de privilegios específicos para proteger la confidencialidad de los datos.
6. **Poblar la base de datos** con información de prueba (datos semilla) realista que permita simular escenarios de uso cotidiano y carga de la operadora.
7. **Diseñar e implementar consultas SQL y vistas** optimizadas que faciliten la rápida generación de reportes estratégicos o analíticos, como el consumo de clientes, rendimiento de servicios y control de los recursos de infraestructura operativa.
